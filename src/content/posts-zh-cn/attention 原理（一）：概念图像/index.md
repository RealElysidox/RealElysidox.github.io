# Attn 之前的遗老们

传统序列处理（2017 之前）的思路可以分成两类：
- **RNN**：逐 word 喂给模型，模型本身是状态机，每个 step 都更新一次状态。
- **CNN**：窗口式遍历，利用卷积提取局部信息。

![Vanilla RNN NLP sequence model](RNN.png)
![Vanilla CNN NLP sequence model](CNN.png)

以现在的眼光来看，它们都没有办法随着算力规模 scaling。RNN 时序让处理时间随着序列长度无限增长，无法同时计算整个序列的信息。CNN 需要让卷积在整个序列上滚转，和 CNN 一样，而且序列变长后感受野不变会引发强归纳偏置，严重过拟合。它们都无法依算力增长按 $O(n)$ 吃红利，直接锁死了技术路径的上限。很多 paper 和 blog 都会这样不厌其烦地告诉你，我们现在算力和数据都够了，Attn 这种拥有全局视野的模型就是能更好地使用 GPU 来加速了 blablabla，所以 Attn 就应该比它们更强。

# Attn 有效的强假设

Attention 的机制是 “先注意上下文和当前位置，算一个最合理的预测”，根据上下文动态计算预测注意力权重，再计算要预测的位置哪些 token 的生成概率最大。但几乎所有的 paper 和 blog 都没有说明，为什么 Attn 在语言模型上有效，而在 RNN 和 CNN 的传统强项时序预测上没有搞出什么新闻。因为很多人做着模型，忘了 Attn 之所以有效的几个强假设，这里我们把它们明确写出来：

1. token 可以进行极高信息密度的离散表示，导致词表可以被经济地训练

实际上，人类的任何语言，词典规模相对于现在的存储数量级那都是极其有限的。汉语只有 40 万个单词和 8.5 万个汉字，而其中常用的单词汉字不会超过 5%；英语单词总量有 100 万，但母语为英语的人通常也仅掌握 2 万到 3 万个单词，而掌握 2000-5000 个高频词即可覆盖大部分日常交流和文章内容。如果我们考虑单词的构词法，把各种派生、前缀、后缀通过更合理的匹配来分割，最终在顶级 LLM 里做出来的词表，最多也不过只有 $10^{6}$ 量级。这个数量看上去很恐怖，但你把它和人类的所有文稿、对话来做比较，大小其实也不值一提了。

单词本身就是极端低熵高信息的对世界上各种物体、概念、逻辑的抽象 embedding，模型训练也需要高效的 Tokenizer（如 BPE 算法）和有限词表的进行数据的强制离散化与语义压缩，将零散的字符或信号组合成高信息密度的表征单元。现代最强的 LLM 也只需要将近 $10^{4}$ 量级的词嵌入维度，早期更是只需要 ~1000 维，这其实恰恰说明了词表的规模让这一切在现在的算力下变得可能。

2. 语法本身就是一种强力的逻辑先验，这让我们只需要有限的上下文就能得到不错的结果

只要学过 position encoding 就会知道，裸 Attn 在数学上是排列不变的（Permutation Invariant），交换 token 顺序不能改变算法的计算结果。这其实反向说明了语言序列带有极其有效的逻辑归纳信息。自然语言的语法规则和固定表达范式，直接为序列施加了强力的马尔可夫约束。

根据信息论测算，人类语言序列的实际香农熵（Shannon Entropy）通常仅为 7 到 10 bits/word，这意味着在给定部分前置上下文的前提下，下一个 token 的在 embedding 空间的概率分布极其稀疏。这种极低的有效信息熵从根本上抹除了 token 之间排列组合的随机性，给了序列强大的结构化先验，引导 Attention 中 $Q K^T$ 矩阵的权重分布自然走向稀疏，并快速收敛于符合语法结构的 token 局部依赖与长程关联上。

![Gemini 估算出的人类和 LLM 实际信息利用率](gemini.png)

因此，在 8K 到 128K 这个数量级的有限上下文窗口内，模型就已经能够捕获充足的条件概率路径来逼近人类语言的宏观逻辑闭环。如果剥离了语法层面的约束，面对完全随机的高熵序列，现在的上下文规模将完全无法支撑 Attn 在算力墙内提取出任何有效特征。

# Attn 的能力边界

基于这些分析我们可以知道，只有当底层数据能被无损或低损耗地压缩成高信息密度、有限规模的离散表征时，Attention 才能在有限上下文内发挥作用。如果任务本身是高熵、连续、或缺乏强语法结构的，强制切分词元不仅会丢失核心信息，还会导致 Attention 陷入纯粹的噪声匹配和维度灾难。以下是几类极其不适合单纯依赖 Attn 机制的典型任务场景：

1. 高频连续时序预测（如金融 Tick 数据、气象与地震波传感数据）
- 边界冲突：这类数据是纯粹的连续实数流，内部充满布朗运动式的随机游走与白噪声。它们不存在类似自然语言的有限词表，也极度缺乏固定的语法约束（低有效信息熵）。
- 强行 Tokenize 的后果：如果强行将连续数值分箱（Binning）转化为离散词元，要么分箱过粗导致关键的微小波动信息丢失，要么分箱过细导致词表爆炸且每个 Token 的出现概率趋于均匀（高熵）。在缺乏结构化先验的情况下，Attention 的全局路由极易拟合到局部随机噪声上，导致预测性能通常不如简单的线性回归模型（如 ARIMA）或擅长提取局部连续特征的 CNN/RNN。

2. 未经预处理的原始物理信号（原始音频波形、像素级视频流）
- 边界冲突：原始物理信号的采样率极高（如 44.1kHz 的音频，意味着每秒 44100 个数据点；或 4K 60fps 的视频）。单个采样点（一个像素或一个声学振幅）的信息密度极低，且自身不具备任何独立的语义。
- 强行 Tokenize 的后果：如果直接以原始采样点作为输入，序列长度 $N$ 会瞬间爆炸，直接撞死在 Attention $O(N^2)$ 的算力墙上；如果不做降采样而直接截断上下文，模型能看到的物理时间窗口将短到无法提取任何宏观特征。
- 行业妥协：目前音视频大模型（如 Sora 或现代语音模型）绝对不敢让 Attention 直接处理原始信号，必须先通过 VAE（变分自编码器）或 VQ（矢量量化）技术，在潜空间（Latent Space）中强行把连续信号打包、聚类成离散的 Patch 或 声学伪词表，这恰恰证明了 Attention 处理连续信号的无力。

3. 高自由度物理系统的连续闭环控制（如复杂机器人运动学控制、高频自动驾驶底层执行）
- 边界冲突：物理世界的连续控制信号（如电机的电流输出、转向角的连续微调）要求极高的平滑性和实时反馈。这类任务不仅状态空间连续，而且其正确性高度依赖于物理定律的偏微分方程约束，而非统计学上的共现概率。
- 强行 Tokenize 的后果：将连续的物理输出动作强制离散化为 Token，会在执行端引入不可接受的量化误差（Quantization Error），导致机械臂产生高频抖动或控制发散。Attention 的全局感受野在这里不仅没有收益，反而会因为计算延迟破坏控制系统的实时闭环。此类任务至今仍是强化学习（结合 MLP 或连续动作空间的 Actor-Critic 架构）乃至传统 PID 控制的统治区。

总的来说，当信息的本质是连续变化、高频振荡且缺乏统计学规律的语法收敛时，强行建立离散词元（Tokenization）属于一种破坏性压缩。在没有高信息密度 Token 作为支撑的荒漠里，Attention 的全局搜索能力毫无用武之地。
